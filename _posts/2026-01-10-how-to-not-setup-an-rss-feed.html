Introduction

You may read the title and say "Wait, you have an RSS feed. Why is the title saying how to NOT set up and RSS feed?"

Well, if you see the commit history, you will understand as to why I say that. It took two (or three) days, about 41 commits, which some did fail to finally get a functioning RSS Feed. There were a lot simpler ways to set up one, but the odds were stacked against me for this one. To understand, we must start from the beginning.

Gentlemen, a short view back to the past...

When I created this website, I had never intended to make a blog. It was only until July 21st, three days since I started the website that I made this blog. You will only understand why I have added this segment if you know how to make a blog on a GitHub Pages website.

To make a blog, you will need a plugin called Jekyll to make your life easier. I never used Jekyll, and probably never will. I am a idiot. A great one, infact. I left the website dormant for a while until this year, when I decided to start posting more *often*, and so decided to set up an RSS feed.

And so it begins

There was this nice, but vague article by Dzhavat Ushev that gave a simple idea as how to set up an RSS feed for your website. It first told you to add Jekyll to your website.

Sure, at GitHub Actions there were options to add Jekyll to your website. I added the one with GitHub Pages dependancies preinstalled. Great! Now was to set it up

I followed the article, and hoped that the feed will come and will bless me in its glorious, hideous XML. Oh was i so wrong. So, so wrong. Commit the changes annnnnnnnd nothing. Absolutely nothing. The feed was there but no post entry. I was stunned. How, what, why? Something had gone wrong, but I don't know what!

The debugging was nothing short of pain - tried one thing, didn't work. Tried another thing, made it worse. I feel like the commit history just spoke for itself. The back and forth commits kept happening and happening until I was getting somewhere

The End

I just kept on poking in the dark like a madman, and I felt the pain will just never end. I reinstalled dependancies, did things a hundred times, until finally. It happened

After pointing Jekyll to the right folder, I pushed the changes and it appeared, beautiful XML showing my blog. I was so happy! The pain was over! I did some minor housekeeping and there we are today!

You can view the feed at https://iswearman.github.io/feed.xml. However, some things did break like viewing the blogposts, but the housekeeping had fixed it

Conclusion

So, what happened?

Jekyll couldn't find the folder where I was keeping the blogposts and so would not load. That was it. It was my first time doing this, so go easy on me

Thank you all for reading! It was stupid on my part, but it happens. I hope you enjoyed the read! I will go and do whatever I want to do next and I will see you next time!

Cheers!
